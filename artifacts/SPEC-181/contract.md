# SPEC-181 Contract — Deterministic postcondition verifier
verifyPostcondition / evalCheck / resolvePath: a serializable postcondition (path+op+value checks: eq/ne/exists/nonempty/gt/gte/lt/lte) verified DETERMINISTICALLY against an observed result — ALL checks must hold ⇒ COMPLETED, else FAILED_FINAL listing the failed checks; a malformed postcondition fails closed. No LLM decides success (INV-01).
Fail-closed (INV-05): when the decision cannot be verified, the safe side is chosen (ASK / DENY / not-usable), never act. Deterministic, no LLM/DB/clock — any time input is injected (INV-01). Result uses the G01 ComponentResult idiom; no boolean success, no thrown errors across the boundary. Rollback: `git revert --no-edit <SPEC-181 commit>` → parent tree MATCH.
