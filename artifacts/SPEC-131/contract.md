# SPEC-131 Contract — Workflow template registry
WorkflowTemplateRegistry + validateTemplate: an immutable, versioned registry of named ordered workflow templates (steps carry action, sideEffect flag, onFailure mode, compensates target); validates unique step ids, existing compensation targets, non-retryable compensating steps, and bounded step count; a new version is a new entry (never an edit), so durable instances pin their version.
Fail-closed (INV-05): when the decision cannot be verified, the safe side is chosen (ASK / DENY / not-usable), never act. Deterministic, no LLM/DB/clock — any time input is injected (INV-01). Result uses the G01 ComponentResult idiom; no boolean success, no thrown errors across the boundary. Rollback: `git revert --no-edit <SPEC-131 commit>` → parent tree MATCH.
