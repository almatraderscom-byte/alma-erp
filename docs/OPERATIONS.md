## ৩. Kill Switch — কীভাবে কী বন্ধ করবেন

| কী বন্ধ করবেন | কোথায় | কীভাবে |
|---|---|---|
| **পুরো এজেন্ট** (চ্যাট, API, সব) | Vercel | Vercel → alma-erp project → Settings → Environment Variables → `AGENT_ENABLED` মুছুন বা `false` করুন → Redeploy |
| **সব স্বয়ংক্রিয় job** (রিমাইন্ডার, নামাজ, রিপোর্ট — সব) | VPS | SSH করে: worker-এর `.env`-এ `SCHEDULERS_ENABLED=false` → `pm2 restart agent-worker` |
| **শুধু নামাজের রিমাইন্ডার (এক ওয়াক্ত/এক দিন)** | Telegram | এজেন্টকে বলুন: "আজ Dhuhr skip করো" বা "৩ দিন নামাজ রিমাইন্ডার বন্ধ রাখো" → Approve card |
| **নামাজের কড়াকড়ি কমানো** | Telegram | এজেন্টকে বলুন: "salah escalation level 1 করো" (১=নরম, ২=স্বাভাবিক, ৩=কড়া) |
| **Worker সাময়িক বন্ধ** | VPS | `pm2 stop agent-worker` (চালু করতে: `pm2 start agent-worker`) |

## ৪. "যখন সমস্যা হবে" — Runbook

### Bot চুপ হয়ে গেছে (মেসেজের উত্তর আসছে না)
VPS-এ SSH করে এই ৩টি কমান্ড পরপর চালান:
```
pm2 status
pm2 logs agent-worker --lines 50
pm2 restart agent-worker
```
১ মিনিট পর Telegram-এ `/today` পাঠিয়ে দেখুন উত্তর আসে কিনা।

### "Worker service(s) silent" সতর্কতা এসেছে (ntfy critical)
মানে: VPS-এর worker ৫+ মিনিট ধরে সাড়া দিচ্ছে না। করণীয়:
```
pm2 restart agent-worker
redis-cli ping        ← উত্তর PONG না এলে: systemctl restart redis
```
এরপরও না চললে VPS পুরো reboot করুন — pm2 নিজে নিজে চালু হবে (pm2 startup আগেই করা আছে; নিশ্চিত হতে: `systemctl is-enabled pm2-root`)।

### এজেন্ট ভুলভাল উত্তর দিচ্ছে / পাগলামি করছে
জরুরি হলে kill switch (উপরের সেকশন ৩) — `AGENT_ENABLED=false`। ERP-এর কোনো ক্ষতি হবে না; ERP সম্পূর্ণ আলাদা চলে।

### সতর্কতার মানে কী
| কী এলো | মানে | কী করবেন |
|---|---|---|
| ntfy সাধারণ টপিক (`alma-agent`) | তথ্যমূলক — রিমাইন্ডার, রিপোর্ট | কিছু না, পড়ুন |
| ntfy critical টপিক (`alma-agent-crit`) | জরুরি — worker down, বাজেট ১০০%, নামাজ শেষ মুহূর্ত | দেখে ব্যবস্থা নিন |
| **ফোন কল (Twilio)** | সর্বোচ্চ জরুরি (দিনে সর্বোচ্চ ২ বার আসতে পারে) | অবশ্যই Telegram খুলে বিস্তারিত দেখুন |

## ৫. মাসিক খরচ

- **কোথায় দেখবেন:** https://alma-erp-six.vercel.app/agent/costs — আজ/এই মাস/পূর্বাভাস, প্রোভাইডার অনুযায়ী ভাগ, CSV ডাউনলোড বাটন।
- **প্রতিদিন রাত ১১:৩০-এর সারাংশে** "আজকের AI খরচ: $X.XX" লাইন থাকে।
- **প্রত্যাশা:** স্বাভাবিক ব্যবহারে মাসে আনুমানিক **$১০–২৫** (মূল খরচ Claude API; ছবি বানালে প্রতি ছবি ৳১.১০–৳৪.৫০, ভয়েস ও ট্রান্সক্রিপশন খুবই সামান্য)। ভারী ব্যবহারে বেশি হতে পারে — তাই বাজেট সেট করুন।
- **বাজেট সতর্কতা:** /agent/costs পেজে দৈনিক ও মাসিক বাজেট (USD) সেট করুন। ৮০% ছুঁলে সাধারণ সতর্কতা, ১০০% ছুঁলে critical সতর্কতা — প্রতি দিনে/মাসে একবারই।

## ৬. Hermes (পুরনো বট) বন্ধ করার চেকলিস্ট

⚠️ **এগুলো এখনো করা হয়নি — আপনি চাইলে তবেই, ধাপে ধাপে করা হবে।** নতুন এজেন্ট কয়েক সপ্তাহ নির্ভরযোগ্যভাবে চলার পর করাই ভালো।

1. ডেটা মাইগ্রেশন আবার যাচাই: `node worker/scripts/migrate-hermes-finance.mjs --dry-run` (নতুন কিছু না দেখালে সব আনা হয়ে গেছে)
2. Hermes VPS-এ পুরনো প্রসেস বন্ধ: `pm2 delete <hermes-process-name>` + `pm2 save`
3. BotFather-এ পুরনো Hermes বট-এর token revoke করুন (`/revoke`) বা বট ডিলিট করুন
4. Hermes-এর SQLite ডাটাবেস আর্কাইভ নিশ্চিত করুন: `/opt/agent-backups/hermes-final/hermes.db`
5. **সবশেষে** (আলাদা অনুমোদনের পর): এই repo থেকে `/api/agent/*` রুট ও `ALMA_AGENT_API_KEY`, IP allowlist (31.97.237.40) সরানো — এটা কোড পরিবর্তন, আলাদা ফেজে হবে
6. Vercel ও VPS থেকে Hermes-সংক্রান্ত পুরনো env var মুছে ফেলা
