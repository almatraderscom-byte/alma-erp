smartOpen() -> pushSmart() (SwiftUIShell.swift:261-268); the audited
"root callbacks call pushWeb directly" defect is already closed on this base.
Remaining pushWeb entry conditions:
  1. query-carrying deep links (pushSmart line 233-237: query => web, documented);
  2. same-origin escape hatch "ওয়েবে খুলুন" (line 246, intentional recursion guard);
  3. router-miss fallback (line 252 + AssistantSwiftUI.swift:7825);
  4. notification tap with query (line 566).
IOSP-1 targets: (1) query deep links should resolve native where the native
screen supports the parameter; (3) unknown internal routes must fail loudly
(structured telemetry) instead of silently embedding web.
